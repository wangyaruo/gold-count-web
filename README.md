# 黄金小账本

黄金小账本是一个本地使用的网页记账工具，用来记录黄金买入、卖出、克数、单价、费用和备注。它会根据行情源获取的当前金价，自动计算当前持仓、剩余成本、估值、持仓浮盈、已实现盈亏和总盈亏。

## 功能概览

- 记录买入和卖出交易。
- 支持日期、克数、单价、费用和备注。
- 支持编辑、删除交易。
- 支持按全部、买入、卖出筛选交易列表。
- 自动获取当前金价，并显示价格来源银行。
- 自动计算持仓克数、剩余成本、当前估值、持仓浮盈、已实现盈亏和总盈亏。
- 通过本地服务把数据保存到 `mock/ledger-data.json`。

## 本地使用

先安装依赖：

```bash
npm install
```

日常使用时启动本地生产服务：

```bash
npm run start
```

这个命令会先执行生产构建，再启动本地服务。浏览器打开终端输出的本地地址即可使用，默认地址通常是：

```text
http://127.0.0.1:4173/
```

如果 4173 端口被占用，服务会自动尝试后面的端口。

macOS 也可以直接双击项目根目录下的 `start-local.command`，它会启动本地服务并自动打开浏览器。

如果已经构建过，只想直接启动已有 `dist` 文件：

```bash
npm run serve:local
```

开发调试时再启动 Vite 开发服务：

```bash
npm run dev
```

页面会通过 `/api/ledger-data` 接口读取和保存账本数据。

## 数据保存

当前版本没有登录、数据库和云端同步。账本数据保存在项目文件：

```text
mock/ledger-data.json
```

新增、编辑、删除交易，以及刷新当前金价时，本地服务会把最新数据写回这个文件。

## 当前金价来源

当前金价会由本地服务从 `gold-analysis-web` 的行情接口获取：

```text
http://154.219.120.25:8318/api/market/snapshot
```

账本使用响应中的 `price.display_value` 作为当前金价，单位为 `CNY/g`，并显示 `price.source` 作为价格来源，例如“工商银行积存金”。如果行情接口暂时不可用，页面会继续使用 `mock/ledger-data.json` 中上次保存的当前金价。

可以通过环境变量覆盖行情服务配置：

```bash
GOLD_ANALYSIS_API_BASE_URL=http://127.0.0.1:8318 npm run start
GOLD_ANALYSIS_API_TOKEN=your-token npm run start
GOLD_ANALYSIS_PRICE_SOURCE=icbc npm run start
```

纯静态托管环境，例如只启用 GitHub Pages，不能把用户操作写回仓库文件。因此静态页面可以展示和运行前端代码，但不能替代本地开发服务完成文件持久化。

## 盈亏计算口径

- 买入成本 = 费用。
- 卖出收入 = 费用。
- 克数、费用、单价会自动联动，任意输入其中两项会补全第三项。
- 已实现盈亏按 FIFO 计算，即最早买入的黄金先被卖出。
- 剩余成本是卖出匹配后仍持有黄金的成本。
- 当前估值 = 当前金价 × 当前持仓克数。
- 持仓浮盈 = 当前估值 - 剩余成本。
- 总盈亏 = 已实现盈亏 + 持仓浮盈。

## 使用提醒

- 当前金价会自动从行情源获取；行情源失败时使用上次保存值。
- “费用”表示本笔交易总金额。
- 卖出克数不能超过当前可用持仓。
