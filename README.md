# 黄金小账本

黄金小账本是一个本地使用的网页记账工具，用来记录黄金买入、卖出、克数、单价、费用和备注。它会根据你手动输入的当前金价，自动计算当前持仓、剩余成本、估值、持仓浮盈、已实现盈亏和总盈亏。

## 功能概览

- 记录买入和卖出交易。
- 支持日期、克数、单价、费用和备注。
- 支持编辑、删除交易。
- 支持按全部、买入、卖出筛选交易列表。
- 手动保存当前金价。
- 自动计算持仓克数、剩余成本、当前估值、持仓浮盈、已实现盈亏和总盈亏。
- 通过本地开发服务把数据保存到 `mock/ledger-data.json`。

## 技术栈

- Vue 3
- TypeScript
- Vite
- Element Plus
- Vitest

## 本地运行

先安装依赖：

```bash
npm install
```

启动本地开发服务：

```bash
npm run dev
```

浏览器打开 Vite 输出的本地地址即可使用。页面会通过 `/api/ledger-data` 接口读取和保存账本数据。

## 数据保存

当前版本没有登录、数据库和云端同步。账本数据保存在项目文件：

```text
mock/ledger-data.json
```

新增、编辑、删除交易，以及保存当前金价时，本地开发服务会把最新数据写回这个文件。

纯静态托管环境，例如只启用 GitHub Pages，不能把用户操作写回仓库文件。因此静态页面可以展示和运行前端代码，但不能替代本地开发服务完成文件持久化。

## 盈亏计算口径

- 买入成本 = 克数 × 买入单价 + 费用。
- 卖出收入 = 克数 × 卖出单价 - 费用。
- 已实现盈亏按 FIFO 计算，即最早买入的黄金先被卖出。
- 剩余成本是卖出匹配后仍持有黄金的成本。
- 当前估值 = 当前金价 × 当前持仓克数。
- 持仓浮盈 = 当前估值 - 剩余成本。
- 总盈亏 = 已实现盈亏 + 持仓浮盈。

## 使用提醒

- 当前金价需要手动输入，不会自动联网获取。
- “费用”表示手续费、工费或溢价；即使费用为 0，买入黄金本身的金额仍会计入剩余成本。
- 卖出克数不能超过当前可用持仓。
