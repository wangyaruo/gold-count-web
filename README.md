# 黄金小账本

一个用于记录黄金买入、卖出、克数、单价和费用的本地网页小账本。它会根据手动输入的当前金价计算持仓估值、持仓浮盈、已实现盈亏和总盈亏。

## 功能

- 记录买入和卖出交易。
- 支持日期、克数、单价、费用和备注。
- 手动保存当前金价。
- 自动计算持仓克数、剩余成本、当前估值、持仓浮盈、已实现盈亏和总盈亏。
- 支持交易编辑、删除和买入/卖出筛选。
- 数据保存在项目的 `mock/ledger-data.json`，刷新页面后仍然保留。

## 技术栈

- Vue 3
- TypeScript
- Vite
- Element Plus
- Vitest

## 本地运行

```bash
npm install
npm run dev
```

浏览器打开 Vite 输出的本地地址即可使用。

本地开发服务会提供 `/api/ledger-data` 接口，把页面上的新增、编辑、删除和当前金价保存到 `mock/ledger-data.json`。

## 盈亏计算口径

- 买入成本 = 克数 × 买入单价 + 费用。
- 卖出收入 = 克数 × 卖出单价 - 费用。
- 已实现盈亏按 FIFO 计算：最早买入的黄金先被卖出。
- 剩余成本是卖出匹配后仍持有黄金的成本。
- 当前估值 = 当前金价 × 当前持仓克数。
- 持仓浮盈 = 当前估值 - 剩余成本。
- 总盈亏 = 已实现盈亏 + 持仓浮盈。

## 数据说明

当前版本没有登录和云端同步，数据文件位于：

```text
mock/ledger-data.json
```

页面需要通过 `npm run dev` 启动的本地服务读写这个文件。纯静态托管环境（例如只开启 GitHub Pages）不能把用户操作写回仓库文件。
